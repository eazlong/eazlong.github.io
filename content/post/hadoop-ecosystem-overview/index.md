## 什么是 Hadoop

Hadoop 是一个能够对大量数据进行**分布式存储与计算**的开源框架。它的核心思想是：与其将数据搬到计算节点，不如将计算逻辑分发到数据所在的节点上执行，从而实现海量数据的并行处理。

---

## 架构分层

Hadoop 生态可以分为**存储层**、**计算引擎层**和**上层应用**三个层次：

```
┌─────────────────────────────────────────────┐
│          上层应用 / 数据库引擎               │
│    HBase (列式数据库)  Cassandra (NoSQL)     │
├─────────────────────────────────────────────┤
│              计算引擎层                      │
│    MapReduce (批处理)   Spark (通用计算)      │
├─────────────────────────────────────────────┤
│              存储层                          │
│           HDFS (分布式文件系统)               │
└─────────────────────────────────────────────┘
```

## 核心组件详解

### 1. HDFS — 分布式文件系统

HDFS（Hadoop Distributed File System）是 Hadoop 的存储基座，负责将文件切分为固定大小的数据块（默认 128MB），分散存储在集群的多个节点上。

| 特性 | 说明 |
|------|------|
| 高容错 | 每个数据块默认 3 副本，节点宕机不丢数据 |
| 高吞吐 | 针对大文件顺序读写优化，适合批处理 |
| 横向扩展 | 增加 DataNode 即可扩展存储容量 |
| 不适合场景 | 低延迟随机读写、大量小文件存储 |

**核心角色**：
- **NameNode**：管理文件系统的元数据（目录树、块映射）
- **DataNode**：存储实际的数据块，执行读写操作

---

### 2. MapReduce — 批处理编程模型

MapReduce 是处理大量半结构化数据的编程模型，将任务拆分为两个阶段：

```
输入数据 → Map（分布处理） → Shuffle（排序分组） → Reduce（归约汇总） → 输出结果
```

**Map 阶段**：将输入数据拆分为键值对，在多个节点上并行处理。

**Reduce 阶段**：将 Map 阶段的中间结果按 key 分组，归约汇总生成最终结果。

**经典示例 — WordCount**：

```
# Map: 每遇到一个单词，输出 (单词, 1)
"hello world hello" → (hello, 1), (world, 1), (hello, 1)

# Reduce: 按 key 归约求和
(hello, [1, 1]) → (hello, 2)
(world, [1])    → (world, 1)
```

> **局限性**：Map 和 Reduce 之间的中间数据需要写入磁盘，I/O 开销大，不适合迭代计算和交互式查询。

---

### 3. Spark — 通用并行计算框架

Spark 是 MapReduce 的升级替代，核心优势在于**内存计算**——中间输出和结果可以保存在内存中，避免反复读写磁盘。

| 对比 | MapReduce | Spark |
|------|-----------|-------|
| 中间结果 | 写入磁盘（HDFS） | 保留在内存（RDD） |
| 迭代计算 | 每次迭代都读写磁盘，极慢 | 内存复用，快 10-100 倍 |
| 编程模型 | 仅 Map + Reduce | 丰富的算子（filter、join、groupBy 等） |
| 适用场景 | 大规模批处理 | 批处理、流处理、机器学习、图计算 |
| 延迟 | 分钟级 | 秒级到分钟级 |

**Spark 生态**：
- **Spark SQL**：结构化数据查询
- **Spark Streaming**：近实时流处理
- **MLlib**：机器学习库
- **GraphX**：图计算引擎

---

### 4. HBase — 分布式列式数据库

HBase 是构建在 HDFS 之上的分布式、面向列的开源数据库，适合海量数据的实时随机读写。

| 特性 | 说明 |
|------|------|
| 数据模型 | 稀疏、分布式、多维有序的 Map |
| 存储方式 | 面向列族（Column Family）存储 |
| 适用场景 | 海量数据的实时随机读写（如用户画像、时序数据） |
| 不适合场景 | 复杂 SQL 查询、多表关联 |

**与传统关系型数据库的区别**：

```
关系型数据库：行存储，适合事务处理（OLTP）
HBase：       列存储，适合海量数据的快速查询（特定列）
```

---

### 5. Cassandra — NoSQL 分布式存储引擎

Cassandra 是一个去中心化的 NoSQL 分布式数据库，最初由 Facebook 开发，特点是**无单点故障**和**线性扩展**。

| 特性 | 说明 |
|------|------|
| 架构 | 去中心化（P2P），无 Master 节点 |
| 可用性 | 支持多数据中心部署，AP 优先（CAP 定理） |
| 数据模型 | 宽列存储（Wide Column Store） |
| 写性能 | 极高，采用 LSM-Tree 结构 |
| 适用场景 | 高写入吞吐、时序数据、IoT、日志存储 |

## 技术选型参考

| 需求 | 推荐方案 |
|------|----------|
| 海量数据批处理 | HDFS + Spark |
| 实时随机读写 | HBase |
| 高写入吞吐 + 多数据中心 | Cassandra |
| 流式数据处理 | Spark Streaming / Flink |
| 交互式 SQL 查询 | Spark SQL / Hive / Presto |
| 机器学习 | Spark MLlib / TensorFlow on YARN |
