## 什么是哈夫曼树

给定 n 个带权节点，构造出**带权路径长度（WPL）最小**的二叉树，称为哈夫曼树（Huffman Tree），也叫**最优二叉树**。

### 关键术语

| 术语 | 含义 |
|------|------|
| **权值** | 节点被赋予的数值（通常表示频率或重要性） |
| **路径长度** | 从根节点到某个节点经过的边数 |
| **带权路径长度（WPL）** | 所有叶子节点的 `权值 × 路径长度` 之和 |

> **目标**：让权值大的节点离根更近（路径更短），权值小的节点离根更远，从而使 WPL 最小。

---

## 构造过程

以节点权值 `{1, 2, 3, 4, 4, 5, 4, 6}` 为例，逐步构造哈夫曼树：

### 步骤

**第 1 步**：将所有节点各自视为一棵只有根节点的树，形成一个"森林"。

```
森林: [1] [2] [3] [4] [4] [5] [4] [6]
```

**第 2 步**：选择权值最小的两棵树，合并为一棵新树，新树的根节点权值 = 两子节点权值之和。

```
选择 1 和 2，合并：
    (3)
   /   \
  1     2

森林: [(3)] [3] [4] [4] [5] [4] [6]
```

**第 3 步**：将新树放回森林，删除已选的两棵树。

**第 4 步**：重复第 2、3 步，直到森林中只剩一棵树。

```
第二轮：选择 3 和 3，合并为 (6)
森林: [(6)] [4] [4] [5] [4] [6]

第三轮：选择 4 和 4，合并为 (8)
森林: [(6)] [(8)] [5] [4] [6]

第四轮：选择 4 和 5，合并为 (9)
森林: [(6)] [(8)] [(9)] [6]

第五轮：选择 6 和 (6)，合并为 (12)
森林: [(12)] [(8)] [(9)]

第六轮：选择 (8) 和 (9)，合并为 (17)
森林: [(12)] [(17)]

第七轮：选择 (12) 和 (17)，合并为 (29)
森林: [(29)]  ← 构造完成
```

### 特点

- 哈夫曼树**没有度为 1 的节点**（每个非叶子节点都有两个子节点）
- n 个叶子节点的哈夫曼树共有 `2n - 1` 个节点
- 构造结果**不唯一**（左右子树可互换），但 WPL 一定最小

## 哈夫曼编码

哈夫曼编码是哈夫曼树最经典的应用场景，用于**无损数据压缩**。

### 核心思想

将字符的出现频率作为权值构造哈夫曼树，使得：
- **高频字符** → 短编码（靠近根节点）
- **低频字符** → 长编码（远离根节点）

### 编码规则

- 从根节点出发，走**左子树记为 0**，走**右子树记为 1**
- 到达叶子节点时，路径上的 0/1 序列即为该字符的编码

### 示例

假设字符频率如下：

| 字符 | 频率 |
|:----:|:----:|
| A | 5 |
| B | 9 |
| C | 12 |
| D | 13 |
| E | 16 |
| F | 45 |

构造哈夫曼树后，得到编码：

```
         (100)
        /     \
      (45)    (55)
       F     /    \
           (25)   (30)
          /    \   /   \
        (12)  (13)(14) (16)
         C     D  / \    E
                 (5)(9)
                  A   B
```

| 字符 | 频率 | 编码 | 位数 |
|:----:|:----:|:----:|:----:|
| F | 45 | `0` | 1 |
| C | 12 | `100` | 3 |
| D | 13 | `101` | 3 |
| A | 5 | `1100` | 4 |
| B | 9 | `1101` | 4 |
| E | 16 | `111` | 3 |

### 为什么是最优的

对比等长编码（每个字符 3 位）：

```
等长编码总位数 = (5+9+12+13+16+45) × 3 = 300 位
哈夫曼编码总位数 = 45×1 + 12×3 + 13×3 + 5×4 + 9×4 + 16×3 = 224 位
```

> 节省了 **25%** 的空间。频率分布越不均匀，压缩效果越好。

### 前缀编码特性

哈夫曼编码是一种**前缀编码**——任何一个字符的编码都不是另一个字符编码的前缀。这保证了解码时不会产生歧义，可以从左到右逐位解码。

## 实际应用

| 应用场景 | 说明 |
|----------|------|
| **文件压缩** | ZIP、GZIP 等压缩格式的底层算法之一 |
| **图像编码** | JPEG 图像压缩中的熵编码阶段 |
| **通信传输** | 减少数据传输量，提高带宽利用率 |
| **HTTP/2 HPACK** | HTTP 头部压缩使用了哈夫曼编码 |

## C++ 实现参考

```cpp
#include <queue>
#include <vector>

struct HuffmanNode {
    int weight;
    HuffmanNode* left;
    HuffmanNode* right;

    HuffmanNode(int w) : weight(w), left(nullptr), right(nullptr) {}
};

// 比较器：小顶堆
struct Compare {
    bool operator()(HuffmanNode* a, HuffmanNode* b) {
        return a->weight > b->weight;
    }
};

HuffmanNode* buildHuffmanTree(const std::vector<int>& weights) {
    std::priority_queue<HuffmanNode*, std::vector<HuffmanNode*>, Compare> pq;

    // 将所有节点加入优先队列
    for (int w : weights) {
        pq.push(new HuffmanNode(w));
    }

    // 反复合并最小的两棵树
    while (pq.size() > 1) {
        HuffmanNode* left = pq.top(); pq.pop();
        HuffmanNode* right = pq.top(); pq.pop();

        HuffmanNode* parent = new HuffmanNode(left->weight + right->weight);
        parent->left = left;
        parent->right = right;

        pq.push(parent);
    }

    return pq.top();  // 根节点
}
```
