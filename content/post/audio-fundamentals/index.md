## 声音如何变成数字

自然界的声音是**连续的模拟信号**（一条平滑的波形）。要让计算机处理声音，需要将模拟信号转换为**离散的数字信号**，这个过程涉及三个关键参数：

```
模拟声波 → [采样] → [量化] → [编码] → 数字音频（PCM）
              ↑         ↑         ↑
           采样率     采样位深    声道数
```

---

## 核心概念

### 1. 采样率（Sample Rate）

每秒钟对模拟信号采集多少次样本，单位为 Hz（赫兹）。采样率越高，能还原的频率范围越广，音质越好。

| 采样率 | 频率范围 | 典型应用 |
|:------:|:--------:|----------|
| 8 kHz | 0 ~ 4 kHz | 电话通话（人声） |
| 16 kHz | 0 ~ 8 kHz | VoIP、语音识别 |
| 22.05 kHz | 0 ~ 11 kHz | AM 广播音质 |
| 44.1 kHz | 0 ~ 22 kHz | CD 音质（覆盖人耳听觉范围） |
| 48 kHz | 0 ~ 24 kHz | DVD、专业音频、视频配音 |
| 96 kHz | 0 ~ 48 kHz | 高保真录音、母带制作 |

> **奈奎斯特定理**：采样率必须至少是信号最高频率的 2 倍，才能无失真地还原原始信号。人耳能听到的最高频率约 20 kHz，所以 CD 采用 44.1 kHz 采样率。

### 2. 采样位深（Bit Depth）

每个采样点用多少位（bit）来表示其振幅值。位深越高，振幅的量化精度越高，动态范围越大。

| 位深 | 量化级数 | 动态范围 | 典型应用 |
|:----:|:--------:|:--------:|----------|
| 8 bit | 256 级 | 48 dB | 早期电话、游戏音效 |
| 16 bit | 65,536 级 | 96 dB | CD 音质标准 |
| 24 bit | 16,777,216 级 | 144 dB | 专业录音、混音 |
| 32 bit float | 浮点精度 | >1500 dB | 专业 DAW 内部处理 |

> **动态范围**：每增加 1 bit，动态范围约增加 6 dB。16 bit 的 96 dB 动态范围已经覆盖了从安静的录音室到摇滚音乐会的响度差异。

### 3. 声道数（Channels）

同时记录几路独立的音频信号：

| 声道 | 说明 | 应用 |
|:----:|------|------|
| 1（单声道） | 一路信号 | 电话、语音通话 |
| 2（立体声） | 左右两路信号 | 音乐、视频 |
| 5.1（环绕声） | 6 路信号 | 电影院、家庭影院 |
| 7.1 | 8 路信号 | 高端家庭影院、游戏 |

## 码率计算

### 公式

```
码率（bps）= 采样率 × 采样位深 × 声道数
数据速率（Byte/s）= 码率 / 8
```

### 计算示例

**示例 1：电话音质 PCM**

```
采样率: 8,000 Hz
位深:   16 bit
声道:   1（单声道）

码率 = 8000 × 16 × 1 = 128,000 bps = 128 kbps
数据速率 = 128,000 / 8 = 16,000 Byte/s = 16 KB/s
1 分钟大小 = 16 × 60 = 960 KB ≈ 0.94 MB
```

**示例 2：CD 音质 PCM**

```
采样率: 44,100 Hz
位深:   16 bit
声道:   2（立体声）

码率 = 44100 × 16 × 2 = 1,411,200 bps ≈ 1411 kbps
数据速率 = 1,411,200 / 8 = 176,400 Byte/s ≈ 172 KB/s
1 分钟大小 = 172 × 60 ≈ 10.1 MB
```

**示例 3：专业录音 PCM**

```
采样率: 96,000 Hz
位深:   24 bit
声道:   2（立体声）

码率 = 96000 × 24 × 2 = 4,608,000 bps ≈ 4608 kbps
数据速率 = 4,608,000 / 8 = 576,000 Byte/s ≈ 563 KB/s
1 分钟大小 = 563 × 60 ≈ 33 MB
```

### 常见规格速查表

| 场景 | 采样率 | 位深 | 声道 | 码率 | 1 分钟大小 |
|------|:------:|:----:|:----:|:----:|:----------:|
| 电话通话 | 8 kHz | 16 bit | 1 | 128 kbps | 0.94 MB |
| VoIP 通话 | 16 kHz | 16 bit | 1 | 256 kbps | 1.88 MB |
| CD 音质 | 44.1 kHz | 16 bit | 2 | 1411 kbps | 10.1 MB |
| DVD 音质 | 48 kHz | 16 bit | 2 | 1536 kbps | 11 MB |
| 专业录音 | 96 kHz | 24 bit | 2 | 4608 kbps | 33 MB |

## PCM 与常见音频格式

PCM（Pulse Code Modulation，脉冲编码调制）是**未压缩**的原始音频数据。实际使用中通常会封装或压缩：

| 格式 | 压缩方式 | 典型码率 | 音质损失 | 说明 |
|------|----------|:--------:|:--------:|------|
| WAV | 无压缩（PCM） | 1411 kbps | 无 | Windows 标准，文件体积大 |
| FLAC | 无损压缩 | ~900 kbps | 无 | 体积约为 WAV 的 50-60% |
| ALAC | 无损压缩 | ~900 kbps | 无 | Apple 无损格式 |
| MP3 | 有损压缩 | 128-320 kbps | 有 | 最通用的有损格式 |
| AAC | 有损压缩 | 128-256 kbps | 有 | MP3 的继任者，同码率更优 |
| OGG | 有损压缩 | 96-500 kbps | 有 | 开源，游戏/Web 常用 |
| Opus | 有损压缩 | 6-510 kbps | 有 | 最先进的编码器，VoIP/WebRTC 标准 |

> **选择建议**：
> - 存档/母带：WAV 或 FLAC
> - 日常听歌：MP3 320kbps 或 AAC 256kbps
> - 网络传输/实时通信：Opus

## 在程序中使用

### FFmpeg 查看音频信息

```bash
ffmpeg -i audio.wav
# 输出: Stream #0:0: Audio: pcm_s16le, 44100 Hz, stereo, s16, 1411 kb/s

ffprobe -show_format -show_streams audio.wav
```

### FFmpeg 格式转换

```bash
# WAV → MP3（320kbps）
ffmpeg -i input.wav -b:a 320k output.mp3

# 修改采样率（重采样到 16kHz）
ffmpeg -i input.wav -ar 16000 output_16k.wav

# 立体声转单声道
ffmpeg -i input.wav -ac 1 output_mono.wav

# WAV → Opus（适合网络传输）
ffmpeg -i input.wav -c:a libopus -b:a 64k output.opus
```
